In 1980, Texas Instruments released the **SN76489 Programmable Sound Generator (PSG)**. It appeared in the BBC Micro, the Sega Master System, the ColecoVision, the Sega Game Gear, and dozens of arcade machines. Three square-wave tone channels plus a noise channel driven by a shift register — that is the complete sound palette of an era.

In this chapter we replicate the SN76489 entirely in Verilog on the iCE40UP5K, pipe its PCM output to a **GY-PCM5102** I2S DAC module, and play back a four-phase music demo that includes a scale, a chord, a percussion pattern, and the opening phrase of *Ode to Joy*. We also explore how the original VHDL reference implementation by Matthew Hagerty can be used directly alongside our Verilog port using the GHDL Yosys plugin.

All source lives under `SN76489/` in the companion repository.

**Tools used:**
- [Icarus Verilog](http://iverilog.icarus.com/) (`iverilog`) — testbench simulation
- [GTKWave](http://gtkwave.sourceforge.net/) — waveform viewer
- [Yosys](https://yosyshq.net/yosys/) + [nextpnr](https://github.com/YosysHQ/nextpnr) — synthesis and place-and-route
- [GHDL](https://github.com/ghdl/ghdl) + [ghdl-yosys-plugin](https://github.com/ghdl/ghdl-yosys-plugin) — optional, for the original VHDL core

**Hardware needed:**
- Humble iCE board (iCE40UP5K, ver0.4)
- GY-PCM5102 I2S DAC module (based on the TI PCM5102A)
- Headphones or powered speakers with a 3.5 mm jack

## The SN76489 at a Glance

The SN76489 generates sound by toggling digital outputs. Each of the three **tone channels** (A, B, C) is driven by a down-counter. When the counter reaches zero it reloads from a 10-bit period register and flips a toggle flip-flop, producing a square wave. The **noise channel** clocks a 15-bit LFSR at a rate derived either from one of three fixed dividers or from Channel C's own output.

All four channels feed a **DAC attenuation table** — sixteen entries, −2 dB per step — and the resulting 12-bit amplitudes are summed into a 14-bit output. The chip has no on-chip DAC in the analogue sense; early home computers connected the digital outputs to a simple R-2R ladder. Our FPGA implementation produces a proper signed 14-bit PCM value that we route directly to an I2S audio DAC.

### Register Map

The SN76489 is programmed through a single 8-bit bus. Each write is one or two bytes depending on whether you are setting the low half of a period register, the high half, or an attenuation value.

**Byte type 1** (bit 7 = 1): `{1, reg[2:0], data[3:0]}` — selects a register and loads the low 4 bits of data. For period registers, these are the four LSBs of the 10-bit period.

**Byte type 2** (bit 7 = 0): `{0, 0, data[5:0]}` — supplies the upper 6 bits to the register last addressed by a type-1 byte.

The register file has eight entries:

| Reg | Name | Contents |
|-----|------|----------|
| R0 | Ch A period | 10-bit tone period (two-byte write) |
| R1 | Ch A attenuation | 4-bit index into DAC table |
| R2 | Ch B period | 10-bit tone period |
| R3 | Ch B attenuation | 4-bit index |
| R4 | Ch C period | 10-bit tone period |
| R5 | Ch C attenuation | 4-bit index |
| R6 | Noise control | bit 2 = feedback mode (0=periodic, 1=white); bits[1:0] = shift rate |
| R7 | Noise attenuation | 4-bit index |

### Tone Frequency Formula

The tone period register sets the output frequency:

$$f = \frac{f_{PSG}}{32 \times N}$$

where $$f_{PSG}$$ is the PSG clock and $$N$$ is the 10-bit period value. At the authentic rate of ≈ 3.579 MHz (NTSC colour-burst ÷ 1), middle C (C4, 261.6 Hz) needs $$N = 428$$. We run at 3.6 MHz (36 MHz system clock ÷ 10), giving $$N \approx 112500 / f_{Hz} - 1$$, so C4 → 429, D4 → 382, E4 → 340, and so on.

## System Architecture

The complete design has four Verilog modules wired together in `top.v`:

```
clk (12 MHz)
  │
  ▼
 pll                → clk_36 (36 MHz)
  │
  ├─ psg_div (/10)  → psg_en (3.6 MHz enable)
  │
  ├─ sn76489_audio  ← psg_ce_n, psg_wr_n, psg_data  (from sequencer)
  │    └──────────  → pcm14s (signed 14-bit)
  │
  ├─ sequencer      → psg_ce_n, psg_wr_n, psg_data
  │
  └─ i2s_tx         ← pcm16 (sign-extended to 16-bit)
       ├── I2S_BCK
       ├── I2S_LRCK
       └── I2S_DATA
```

The PLL takes the 12 MHz system clock from the RP2040 and produces 36 MHz. A four-bit divider then fires `psg_en` every ten clock cycles, giving the PSG a 3.6 MHz clock enable. The sequencer holds a ROM of register writes and sends them to the PSG core. The core produces a signed 14-bit PCM mix (`pcm14s_o`), which is sign-extended to 16 bits and fed to the I2S transmitter.

## The SN76489 Core (`sn76489_audio.v`)

The core is a Verilog port of the VHDL reference implementation by Matthew Hagerty (dnotq.io), released under a 3-clause BSD licence. Every register, flip-flop, and combinational process from the VHDL is preserved; only the syntax changes (VHDL `signal _r` → Verilog `reg`, VHDL `signal _x` → Verilog `wire`).

The module interface:

```verilog
module sn76489_audio #(
    parameter FAST_IO_G        = 1'b0,  // 0=32-cycle I/O, 1=~2-cycle I/O
    parameter MIN_PERIOD_CNT_G = 6      // suppress tones above ~19 kHz
) (
    input  wire        clk_i,
    input  wire        en_clk_psg_i,   // 3.6 MHz clock enable
    input  wire        ce_n_i,         // chip enable, active low
    input  wire        wr_n_i,         // write enable, active low
    output wire        ready_o,        // low while write in progress
    input  wire [7:0]  data_i,
    output wire [11:0] ch_a_o,         // unsigned 12-bit channel outputs
    output wire [11:0] ch_b_o,
    output wire [11:0] ch_c_o,
    output wire [11:0] noise_o,
    output wire [13:0] mix_audio_o,    // unsigned 14-bit sum
    output wire [13:0] pcm14s_o        // signed 14-bit PCM sum
);
```

We instantiate it with `FAST_IO_G=1` and `MIN_PERIOD_CNT_G=6`.

### I/O FSM

Writing a byte to the original SN76489 chip required holding `CE_N` and `WR_N` low for 32 PSG clock cycles — a slow process designed around 1970s bus timings. The `FAST_IO_G=1` parameter collapses this to approximately two PSG cycles, which is fine because our sequencer is the only bus master and can wait for `ready_o`.

The FSM has three states: `IO_IDLE` (waiting for `CE_N` and `WR_N` both low), `IO_OP` (counting down, then asserting `en_reg_wr_s`), and `IO_WAIT` (holding until `WR_N` goes high again).

```verilog
// Simplified FSM (FAST_IO_G=1 path)
IO_IDLE: if (ce_n_r==0 && wr_n_r==0) begin
    io_state_x = IO_OP;
    ready_x    = 0;
    io_cnt_x   = 0;     // immediate — no 32-cycle delay
end

IO_OP: begin
    en_reg_wr_s = 1;    // write the register now
    io_state_x  = IO_WAIT;
    ready_x     = 1;
end

IO_WAIT: if (wr_n_r==1) io_state_x = IO_IDLE;
```

All FSM registers are clocked only when `en_clk_psg_i` is high, so the entire state machine runs at the PSG rate even though the system clock is faster.

### Register File and DAC ROM

The register file decodes the two-byte protocol. When a type-1 byte arrives it latches the three-bit register address from bits [6:4] and the four low-period bits from bits [3:0]. When a type-2 byte arrives it delivers the upper six period bits to the previously selected register.

Attenuation writes do a table lookup at write time rather than at output time. A 16-entry ROM maps 4-bit attenuation codes to 12-bit amplitude values. The entries follow the −2 dB per step rule; entry 15 is always zero (silence):

```verilog
dacrom_ar[ 0] = 12'hFFF;   //  0 dB — full volume
dacrom_ar[ 1] = 12'hCB5;   // -2 dB
dacrom_ar[ 2] = 12'hA18;   // -4 dB
// ...
dacrom_ar[15] = 12'h000;   // -∞ dB — silence
```

Because the lookup is done at write time, the channel level registers already hold the final 12-bit amplitude, not the 4-bit attenuation index.

### Tone Counters

Each tone channel has a 10-bit down-counter. A secondary enable `en_cnt_r` pulses once every 16 PSG clock cycles (the PSG clock is itself a clock-enable on the 36 MHz system clock, so this is the equivalent of dividing by 16). On each pulse the counter decrements; when it reaches zero it reloads from the period register and the toggle flip-flop inverts:

```verilog
wire [9:0] ch_a_cnt_x =
    (en_cnt_x && ch_a_cnt_r==0) ? ch_a_period_r :   // reload
    en_cnt_r                    ? ch_a_cnt_r - 1 :   // count
                                  ch_a_cnt_r;         // hold

wire tone_a_x =
    flatline_a_s                     ? 1'b1 :         // too fast — hold high
    (en_cnt_x && ch_a_cnt_r==0)      ? ~tone_a_r :   // toggle
                                        tone_a_r;
```

The `flatline` condition fires when the period is non-zero but smaller than `MIN_PERIOD_CNT_G`. Frequencies that would exceed the Nyquist limit of the 46.875 kHz sample rate are clamped to DC (tone FF held high) rather than aliasing back into the audio band. Amplitude modulation via the attenuation register still works normally while flatline is active.

### Noise Channel

The noise channel uses a **15-bit Galois LFSR** with taps at bit positions 0 and 1:

```
feedback = (noise_ctrl ? lfsr[1] : 0) XOR lfsr[0]
lfsr     = {feedback, lfsr[14:1]}       // right-shift, feedback into MSB
```

When `noise_ctrl=0` (periodic mode), only `lfsr[0]` feeds back, creating a single 1-bit that circulates — producing a repeating tone at the LFSR clock rate. When `noise_ctrl=1` (white noise), XOR-ing bit 1 in as well makes the sequence pseudo-random.

The LFSR is clocked on the rising edge of a noise clock derived from the PSG enable and a second divider. The `noise_shift` field selects among three fixed rates (÷512, ÷1024, ÷2048) or Channel C's own toggle output.

Writing to register R6 resets the LFSR to `15'b100_0000_0000_0000`, preventing lock-up in the all-zeros state.

### Signed PCM Output

The channel outputs are 12-bit unsigned amplitudes. To produce a signed PCM stream suitable for I2S we convert each channel to a ±half-amplitude square wave centred on zero, then sum:

- **Tone high** (toggle FF = 1): output `+level/2` → `{0, level[11:1]}`
- **Tone low** (toggle FF = 0): output `−level/2` → two's complement of the above
- **Flatline active**: output `level − 0x800` (DC offset centred on zero, so AM still works)

Each 12-bit signed value is sign-extended to 14 bits and the four channels are summed:

```verilog
pcm14s_r <= ({{2{sign_a_r[11]}}, sign_a_r} +
             {{2{sign_b_r[11]}}, sign_b_r} +
             {{2{sign_c_r[11]}}, sign_c_r} +
             {{2{sign_n_r[11]}}, sign_n_r});
```

All of this runs on `en_clk_psg_i`, so the PCM value updates at 3.6 MHz — far above the I2S sample rate of 46.875 kHz. The I2S transmitter latches a new sample at each word-clock edge and holds it for the full 64-BCLK frame.

## I2S Transmitter (`i2s_tx.v`)

Inter-IC Sound (I2S) is a synchronous serial protocol designed for digital audio. Three wires carry the audio: BCLK (bit clock), LRCLK (word select / left-right clock), and DATA (serial samples, MSB first).

Our transmitter targets the GY-PCM5102, which uses the PCM5102A DAC from Texas Instruments. It operates in **SCK-less mode**: the serial bit clock (BCK) is also used to regenerate the master clock internally, so the SCK pin is tied to GND.

### Clocking

From the 36 MHz system clock we derive:

- **BCLK** = 36 MHz ÷ 12 = **3 MHz** (toggled every 6 system clocks)
- **LRCLK** = 3 MHz ÷ 64 = **46,875 Hz** (one 64-BCLK frame per sample)

Each frame carries 32 BCLK cycles for the left channel and 32 for the right. We transmit 16 data bits followed by 16 zeros to fill the 32-bit slot. The GY-PCM5102 ignores the padding bits.

### Frame Timing

Standard I2S delays the MSB by one BCLK cycle after the LRCLK edge:

```
LRCLK   ‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾\________________________/‾‾‾
BCLK    _‾_‾_‾_‾_‾_‾_‾_‾_‾_‾_‾_‾_‾_‾_‾_‾...
DATA    ──[delay]──[b15][b14][b13]...[b0][0][0]...
                    ↑ first data bit, 1 BCLK after edge
```

The shift register is loaded at bit_cnt=0 (the delay slot), then shifted out MSB-first starting at bit_cnt=1. Because we output the same mono sample on both channels, the right-channel slot (bit_cnt 32–47) is a copy of the left.

## Demo Sequencer (`sequencer.v`)

The sequencer is a ROM-driven FSM that sends a scripted sequence of register writes to the PSG core.

### ROM Format

Each ROM entry is 32 bits: `{wait_ticks[23:0], byte[7:0]}`. The `wait_ticks` field specifies how many PSG-enable pulses to count before sending `byte` to the chip. Because `psg_en` fires at 3.6 MHz, timing in ticks converts directly to milliseconds: 3600 ticks ≈ 1 ms.

```verilog
localparam NUM_STEPS = 83;
localparam LOOP_STEP = 4;   // loop back here (skip one-time init)

reg [31:0] rom [0:NUM_STEPS-1];
```

### Four Demo Phases

**Phase 1 — C major scale (steps 4–20):** Channel A climbs from C4 to C5, one quarter-note per step (400 ms = 1,440,000 ticks). Each note needs two bytes: a type-1 byte with the low four bits of the period, followed immediately by a type-2 byte with the upper six bits.

To encode C4 (period = 429 = 10'h1AD):
- byte1: `{1, 000, 1101}` = `0x8D` (register R0, low bits = 4'hD)
- byte2: `{0, 0, 011010}` = `0x1A` (upper bits = 6'h1A)

**Phase 2 — C major chord (steps 21–32):** All three tone channels play simultaneously — Channel A = C4 (period 429), B = E4 (period 340), C = G4 (period 286) — held for two seconds.

**Phase 3 — Noise drum pattern (steps 33–50):** Eight white-noise hits (R6 = `0xE4`) followed by eight periodic-noise hits (R6 = `0xE0`). Each hit is a 50 ms burst at full volume (attenuation 0) followed by silence (attenuation 15). The different timbre between white and periodic noise is immediately audible.

**Phase 4 — Ode to Joy (steps 51–82):** The first four bars of Beethoven's melody on Channel A at ≈ BPM 100. A quarter note is 600 ms = 2,160,000 ticks; the final bar includes a dotted-quarter at 900 ms = 3,240,000 ticks. After the phrase, Channel A is silenced and the sequencer loops back to step 4 (Phase 1).

### Bus Handshaking FSM

The sequencer has its own four-state FSM to correctly drive the PSG bus:

```
S_WAIT    → count PSG-enable ticks until cur_wait is reached
S_DRIVE   → assert psg_ce_n=0 and psg_wr_n=0 with the byte
S_READY   → wait for psg_ready=1
S_DEASSERT → deassert chip enable, advance step counter
```

Gating writes through `psg_ready` means the sequencer automatically adapts to either `FAST_IO_G` mode: with fast I/O, `ready` returns in about two PSG cycles; with slow I/O, it returns after 32 cycles. Either way the sequencer never races with the PSG internal FSM.

## Top Level (`top.v`)

`top.v` wires everything together and handles the PLL, reset, PSG clock divider, and sign extension.

### PLL and Reset

The PLL (generated by `icepll`) takes 12 MHz in and produces 36 MHz out. A reset counter holds the design in reset until the PLL locks and the counter saturates:

```verilog
reg [7:0] rst_cnt = 0;
wire resetn = &rst_cnt;   // all ones → released from reset

always @(posedge clk_36)
    if (!pll_locked) rst_cnt <= 0;
    else if (!resetn) rst_cnt <= rst_cnt + 1;
```

`&rst_cnt` is a reduction AND — true only when all 8 bits are 1, i.e. after 255 cycles of the PLL being locked.

### PSG Clock Enable

The PSG clock enable is generated by a simple modulo-10 counter on the 36 MHz clock:

```verilog
always @(posedge clk_36) begin
    psg_en <= 0;
    if (psg_div == 4'd9) begin
        psg_div <= 0;
        psg_en  <= 1;
    end else
        psg_div <= psg_div + 1;
end
```

This fires `psg_en` for exactly one 36 MHz cycle every ten cycles, giving an effective PSG clock rate of 3.6 MHz. The `en_clk_psg_i` input of `sn76489_audio` is specifically designed to accept a single-cycle enable rather than a divided clock, which keeps all registers on a single clock domain and avoids CDC issues.

### Sign Extension

`pcm14s_o` from the PSG core is 14 bits wide. Bit 13 is the sign bit. To fill the 16-bit I2S sample we sign-extend two more bits:

```verilog
wire [15:0] pcm16 = {{2{pcm14s[13]}}, pcm14s};
```

Both left and right I2S channels carry the same mono mix.

## Hardware Wiring

Connect a GY-PCM5102 module to PMOD1B (the B-side of PMOD1):

| GY-PCM5102 pin | PMOD1B pin | FPGA pin | Signal |
|---------------|-----------|----------|--------|
| DIN | P1B1 | 46 | I2S_DATA |
| LCK | P1B2 | 47 | I2S_LRCK |
| BCK | P1B3 | 48 | I2S_BCK |
| VCC | — | 3.3 V | power |
| GND | — | GND | ground |
| FMT | — | GND | I2S format select |
| SCK | — | GND | SCK-less mode |
| XMT | — | open or 3.3 V | unmute |

On the GY-PCM5102 module, FMT and SCK are usually on a separate header. Tie both to GND with short jumper wires. Leave XMT open (the module has an internal pull-up) or connect it to 3.3 V.

## Building and Running

```bash
cd SN76489

# Build the demo (Verilog port of the SN76489 core)
make

# Flash to Humble iCE
make flash

# Optional: build using the original VHDL core (requires GHDL + plugin)
make orig_core
make flash_orig

# Run testbenches
make sim
```

After flashing, the blue LED will blink at about 0.5 Hz to show the design is running. Connect headphones or speakers to the GY-PCM5102 audio jack and you should hear the four-phase music loop: scale, chord, drums, Ode to Joy.

### Using the Original VHDL Core

The repository also includes the original `sn76489_audio.vhd` by Matthew Hagerty under `core/`. The `orig_core` Makefile target uses the GHDL Yosys plugin to load it directly:

```makefile
$(PROJ_ORIG).json: $(VHD_CORE) $(V_SRC_ORIG) core/sn76489_audio_shim.v
    yosys -m ghdl -p " \
        ghdl $(GHDL_PFLAG) --std=08 -gFAST_IO_G='1' -gMIN_PERIOD_CNT_G=6 \
            $(VHD_CORE) -e $(VHD_ENTITY); \
        rename $(VHD_ENTITY) $(VHD_ENTITY)_vhdl; \
        read_verilog core/sn76489_audio_shim.v $(V_SRC_ORIG); \
        hierarchy -top $(TOP); \
        synth_ice40 -top $(TOP) -json $@ \
    "
```

Because the GHDL plugin bakes generic parameters in during elaboration, the resulting Yosys module is non-parametric. The shim `core/sn76489_audio_shim.v` wraps it with the same port list and parameter declarations as the Verilog port, so `top.v` compiles unchanged either way.

This approach is a practical technique whenever you want to mix VHDL and Verilog in a Yosys flow: elaborate the VHDL with GHDL, rename the module, then provide a Verilog wrapper that bridges into it.

## Simulation

Two testbenches are included.

`tb/tb_sequencer.v` exercises the sequencer FSM against a behavioural model of the PSG bus — it dumps a VCD file you can inspect in GTKWave to verify the wait counts and byte ordering without needing hardware.

`tb/tb_pcm16_ext.v` is a standalone check of the sign-extension logic, confirming that positive, negative, and zero 14-bit values map to the correct 16-bit signed values.

Run both with `make sim`.

## What to Try Next

The sequencer ROM is fully data-driven, so you can change the music without touching the FSM. Try encoding a different melody by computing period values from the formula $$N = 112500 / f_{Hz} - 1$$ and filling in new ROM entries with appropriate wait counts.

The register interface is also compatible with original SN76489 software. Period values and attenuation codes from vintage game music files can be played back directly by the sequencer FSM. The `noise_shift_r=2'b11` mode (Channel C as the noise clock) enables classic tonal percussion sounds found in Sega Master System games: set Channel C to a bass frequency, silence its attenuation, and let the noise channel track it.

For amplitude effects, write varying attenuation values on successive PSG cycles — because the DAC lookup runs at 3.6 MHz, smooth volume envelopes are possible with nothing more than a counter driving the attenuation register.
